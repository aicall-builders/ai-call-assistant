## 2. 현재 AWS 구성

### 사용 region
`ap-northeast-2` (서울)

### 사용 중인 AWS 서비스
Lambda · API Gateway(REST) · RDS(MySQL) · ElastiCache(Redis) · S3 · CloudFront · Secrets Manager · KMS · VPC(서브넷/NAT/SG/VPC Endpoint) · EventBridge(스케줄) · CloudWatch · IAM · Lambda Layers · EC2(Grafana 모니터링용)

**외부 서비스(AWS 아님):** CLOVA Speech(Naver Cloud) · OpenAI GPT-4o-mini · Firebase(Auth/FCM) · Solapi(SMS) · OAuth(Kakao/Google/Naver)

### 전체 연결 구조

```
[웹]   사용자 → CloudFront(dk1k75g0ji3vw.cloudfront.net) → S3(웹 버킷)

[API]  Android / Web
          → API Gateway (REST, id: avrq2kzfp9, stage: prod)
          → Lambda (auth / call / nlp)
              ├─ RDS MySQL (call_recorder)
              ├─ ElastiCache Redis (캐시/핫리로드)
              ├─ S3 (오디오 버킷, presigned URL)
              ├─ Secrets Manager (DB 비밀번호)
              └─ KMS (캘린더 토큰 암호화)

[NLP]  nlp_handler → CLOVA STT(Naver) → OpenAI GPT-4o-mini 요약
[스케줄] EventBridge → 폴링 Lambda (CLOVA 콜백 유실 대비)
[알림]  Solapi → SMS 발송
```

### 정상 작동 기능
통화 업로드 → STT → NLP 요약 파이프라인, 전화번호 추출/누적(caller_stats), 고객 관리, 웹 캘린더, OAuth 로그인, GitHub Actions CI/CD, Redis 캐시

### 미해결 / 주의 (인계 핵심)
- **CLOVA STT 간헐적 timeout** → Lambda를 NAT 라우팅되는 **private-subnet-1** 연결로 해결. NAT 안 타는 서브넷에 붙이면 재발.
- **CLOVA 웹훅 콜백 유실** 가능 → 폴링(`STT_MAX_RETRY`/`STT_STALE_MINUTES`)으로 보완 중.
- **삼성 통화녹음 파일 경로 변동** → Android에서 Media Store 권한/경로 다중 처리 필요.
- ⚠️ **`DB_PASSWORD`가 Lambda 환경변수에 평문** → 새 환경에서 재설정 + Secrets Manager 이관 권장.
- ⚠️ **RDS `Publicly Accessible: True`** → 운영 전 private로 막기.
- `ssl_disabled=True` (DB) — 운영 전 SSL 켜기.
- CORS(`CORS_ALLOWED_ORIGINS`) — 운영 전 origin 좁히기.
- **API Gateway ID 재확인 필요** — 덤프 시 `avrq2kzfp9`로 리소스가 안 잡힘. 4장 명령으로 현재 살아있는 API ID 먼저 확정.

## 부록 A. 자동 추출 스크립트

> 새 담당자 PC(AWS CLI 로그인 상태)에서 실행하면 대부분 값을 한 파일로 덤프합니다.
> **secret 값은 출력하지 않습니다(이름만).** 결과를 그대로 공유해도 안전합니다.

```bash
#!/usr/bin/env bash
REGION=ap-northeast-2
exec > handoff_dump.txt 2>&1

echo "===== ACCOUNT / REGION ====="
aws sts get-caller-identity; echo "REGION=$REGION"

echo "===== LAMBDA ====="
aws lambda list-functions --region $REGION \
  --query 'Functions[].{Name:FunctionName,Runtime:Runtime,Handler:Handler,Timeout:Timeout,Memory:MemorySize}' --output table
for FN in call-recorder-api-auth call-recorder-api-call call-recorder-api-nlp call-recorder-api-calendar; do
  echo "--- $FN env(NAMES only) ---"
  aws lambda get-function-configuration --region $REGION --function-name "$FN" \
    --query 'keys(Environment.Variables)' --output json 2>/dev/null
  echo "--- $FN layers ---"
  aws lambda get-function-configuration --region $REGION --function-name "$FN" \
    --query 'Layers[].Arn' --output json 2>/dev/null
done

echo "===== API GATEWAY (REST + HTTP 목록) ====="
aws apigateway get-rest-apis --region $REGION \
  --query 'items[].{id:id,name:name}' --output table
aws apigatewayv2 get-apis --region $REGION \
  --query 'Items[].{id:ApiId,name:Name,endpoint:ApiEndpoint}' --output table

echo "===== RDS ====="
aws rds describe-db-instances --region $REGION \
  --query 'DBInstances[].{Id:DBInstanceIdentifier,Engine:Engine,Ver:EngineVersion,DB:DBName,Public:PubliclyAccessible}' --output table

echo "===== S3 ====="
aws s3 ls

echo "===== CLOUDFRONT ====="
aws cloudfront list-distributions \
  --query 'DistributionList.Items[].{Id:Id,Domain:DomainName}' --output table

echo "===== VPC / SG / NAT / ENDPOINT ====="
aws ec2 describe-security-groups --region $REGION \
  --query 'SecurityGroups[].{Id:GroupId,Name:GroupName}' --output table
aws ec2 describe-nat-gateways --region $REGION \
  --query 'NatGateways[].{Id:NatGatewayId,State:State}' --output table
aws ec2 describe-vpc-endpoints --region $REGION \
  --query 'VpcEndpoints[].{Id:VpcEndpointId,Svc:ServiceName,Type:VpcEndpointType}' --output table

echo "===== SECRETS (NAMES only, 값 없음) ====="
aws secretsmanager list-secrets --region $REGION --query 'SecretList[].Name' --output json

echo "DONE -> handoff_dump.txt"
```
